3. Project: WorkDash (작업 대시보드)

[ 문제 정의 ]

Windows 환경에서 다수의 애플리케이션과 브라우저 탭을 동시에 실행하며 작업할 때, 현재 열려있는 창의 맥락을 한눈에 파악하고 분류하기 어렵다는 문제가 있었습니다. 이로 인해 작업 간 전환(Context Switching)에 불필요한 시간이 소요되고, 원하는 정보나 창을 다시 찾는 데 비효율이 발생했습니다.

목표: 이 '작업 대시보드' 프로젝트의 목표는, 현재 실행 중인 모든 윈도우와 탭을 한곳에서 통합 관리하고, AI를 활용해 자동으로 분류/요약함으로써 사용자가 자신의 작업을 쉽게 탐색하고 관리할 수 있는 환경을 제공하는 것이었습니다.

[ 아키텍처 / 설계 ]

핵심 로직은 Python을 기반으로 설계되었습니다.

  1. (수집) Python 스크립트가 로컬 Windows 환경에서 현재 활성화된 애플리케이션 목록과 브라우저 탭 정보(창 제목, URL)를 실시간으로 수집합니다.
  2. (분석) 수집된 텍스트 데이터(창 제목 묶음)는 Gemini API로 전송됩니다.
  3. (처리) Gemini API는 이 정보를 분석하여, 각 작업을 '업무', '학습', '여가' 등으로 자동 분류하거나 현재 작업 내용을 요약합니다.
  4. (시각화) 이 결과가 Python으로 제작된 데스크톱 GUI 애플리케이션에 표시되어, 사용자는 수동 개입 없이 정리된 작업 목록을 확인할 수 있습니다.

[ 핵심 기능 1: 브라우저 확장 프로그램과 Python을 연동한 작업 데이터 통합 수집 ]

로컬 PC의 작업 데이터를 수집하기 위해 두 가지 파트로 나누어 개발했습니다.

  1. 브라우저 탭 수집: JavaScript 기반의 임시 브라우저 확장 프로그램(Extension)을 제작했습니다. 이 확장 프로그램은 사용자가 열어둔 모든 브라우저 탭의 제목과 URL을 실시간으로 읽어옵니다.
  2. 데이터 통신: 확장 프로그램은 수집한 탭 정보를 JSON 형식으로 파이썬 백엔드 스크립트에 전달합니다.
  3. 데이터 통합: Python 스크립트는 전달받은 JSON 데이터(브라우저 탭)와 Windows API로 수집한 로컬 애플리케이션(창 제목) 목록을 하나로 통합하여 분석을 위한 원본 데이터를 완성합니다.

[ 핵심 기능 2: Gemini API를 활용한 작업 자동 분류 및 요약 ]

[핵심 기능 1]에서 통합 수집된 텍스트 데이터(창 제목, 탭 제목 등)를 Gemini API로 전송했습니다. API가 이 텍스트 묶음을 분석하여 현재 사용자의 작업 맥락을 이해하도록 프롬프트를 설계했습니다. API는 "['업무: 3개', '학습: 2개', '여가: 1개']"와 같이 각 항목을 자동으로 분류하거나, "현재 Python 프로젝트 개발과 React 관련 자료 조사를 병행하고 있음"처럼 현재 작업 상태를 한 문장으로 요약한 결과를 JSON 형식으로 반환합니다. 이 결과는 즉시 대시보드 UI에 시각화됩니다.

[ 나의 역할 및 기여 ]

이 프로젝트는 1인 프로젝트로, 기획부터 설계, 개발까지 모든 과정을 주도했습니다.

  • 프로젝트 아키텍처 설계 및 1인 개발 총괄
  • Python 기반 데스크톱 애플리케이션 UI(GUI) 설계 및 개발
  • Windows 환경의 활성 애플리케이션 정보 수집 로직 구현 (Python)
  • 브라우저 탭 정보 수집을 위한 JavaScript 기반 확장 프로그램 개발
  • 확장 프로그램과 Python 애플리케이션 간의 JSON 데이터 통신 파이프라인 구축
  • Gemini API 연동 및 작업 자동 분류/요약을 위한 프롬프트 엔지니어링

[ 회고 (배운 점) ]

  • (기술 통합) Python 데스크톱 GUI, JavaScript 기반 브라우저 확장 프로그램, 그리고 Gemini라는 외부 API를 연동하는 과정에서 서로 다른 환경 간의 데이터 통신(JSON) 파이프라인을 설계하고 디버깅하는 실전 경험을 쌓았습니다.
  • (AI 활용) 단순히 AI API를 호출하는 것을 넘어, 내가 원하는 '분류'와 '요약' 결과를 정확히 얻어내기 위한 프롬프트 엔지니어링의 중요성을 깊이 이해하게 되었습니다.
  • (문제 해결) 로컬 PC 환경의 데이터를 다루다 보니 다양한 예외 상황(예: 프로그램 응답 없음, 탭 정보 로드 실패)을 마주했습니다. 이를 안정적으로 처리하는 로직을 고민하며 방어적인 코드(Defensive Code) 작성의 필요성을 배웠습니다.

[ 회고 (개선점) ]

  • (성능 최적화) 현재는 일정 주기마다 모든 정보를 새로고침하는데, 이로 인해 시스템에 불필요한 부하가 발생할 수 있습니다. 향후에는 이벤트 기반(Event-driven)으로 변경 사항이 생길 때만 데이터를 수집하도록 최적화하고 싶습니다.
  • (UI/UX 고도화) 지금은 기능 구현에 집중했지만, 사용자가 분류 결과를 직접 수정하거나 중요 작업을 고정하는 등 상호작용이 가능한 UI/UX를 추가하여 사용성을 높이고 싶습니다.
  • (확장성) 현재는 Windows와 특정 브라우저로 한정되어 있습니다. 향후에는 macOS나 Linux 환경, 그리고 Firefox, Edge 등 다른 브라우저도 지원할 수 있도록 구조를 개선해 보고 싶습니다.
